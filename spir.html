SPIr: Segment Pivotality Index for Hierarchical Electoral Data

SPIr is an R package designed to compute and analyse the Segment Pivotality Index (SPI), a diagnostic tool for hierarchical electoral systems such as Assembly Constituencies nested within Parliamentary Constituencies. SPI quantifies how pivotal each sub-unit is to the final election outcome using only meso-level data.

Although created for the Indian electoral context, SPIr is fully generalisable to any hierarchical dataset (e.g., branches within firms, districts within regions).

Installation

Install the development version from GitHub:

install.packages("remotes")
remotes::install_github("PawasPratikshit/SPIr")
library(SPIr)

Documentation & Help

On some systems, package-level help may not open via ?SPIr. The reliable way to browse the full documentation is: 

help(package = "SPIr")

Function-specific help works normally:

?compute_spi
?add_spi
?summary_spi
?plot_spi_distribution

Core Functions

  • compute_spi() – Compute SPI at the segment level
  • add_spi() – Merge SPI values into your dataset
  • summary_spi() – Summarise SPI patterns
  • plot_spi_distribution() – Visualise SPI values
  • spi_help() – Quick workflow guide

Example

library(SPIr)
library(dplyr)

df <- tibble(
  pc    = c(1,1,1,1),
  ac    = c(1,1,2,2),
  cand  = c("A","B","A","B"),
  votes = c(60,40,55,45)
)

spi_df <- compute_spi(
  df,
  pc_id     = pc,
  ac_id     = ac,
  candidate = cand,
  votes     = votes
)

spi_df

Recommended Workflow

help(package = "SPIr")
spi_help()

spi_df  <- compute_spi(...)
full_df <- add_spi(...)
summary_spi(spi_df)
plot_spi_distribution(spi_df)
